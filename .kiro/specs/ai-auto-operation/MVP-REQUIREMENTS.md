# AI创作助手 - MVP需求文档

## 简介

AI创作助手MVP版本是一个面向内容创作者的AI辅助创作工具。系统提供从选题策划、AI文案生成到数字人视频制作的完整创作链路，帮助创作者快速生成高质量的短视频内容。

**核心价值**: 5分钟完成一条短视频内容的创作（选题→文案→数字人视频）

## 术语表

- **系统**: AI创作助手
- **内容创作者**: 使用系统进行内容生产的个人用户
- **选题**: 内容创作的主题和方向
- **AI文案**: 由大模型生成的短视频文案
- **数字人视频**: 使用数字人形象和AI配音生成的视频
- **积分**: 用户使用AI功能消耗的虚拟货币
- **会员套餐**: 系统提供的付费订阅计划

## MVP需求列表

### 需求 1: 用户注册和登录

**用户故事**: 作为内容创作者，我想要注册和登录系统，以便安全地访问我的账号和使用平台功能。

#### 验收标准

1. WHEN 新用户访问注册页面 THEN 系统应显示注册表单，包含邮箱、密码输入字段
2. WHEN 用户提交有效的注册信息 THEN 系统应创建新账号并自动登录
3. WHEN 已注册用户输入正确的登录凭证 THEN 系统应验证身份并创建会话令牌
4. WHEN 用户输入错误的登录凭证 THEN 系统应显示错误提示
5. WHEN 用户登录成功 THEN 系统应跳转到首页并显示用户信息

**MVP简化说明**: 
- 不需要邮箱验证（直接注册即可使用）
- 不需要第三方登录
- 不需要找回密码功能（V2.0添加）

---

### 需求 2: 会员套餐和支付

**用户故事**: 作为内容创作者，我想要购买会员套餐，以便获得更多的积分使用AI功能。

#### 验收标准

1. WHEN 用户访问套餐页面 THEN 系统应显示2个会员套餐（免费版、基础版），包含功能对比和价格信息
2. WHEN 用户选择基础版并点击购买 THEN 系统应跳转到支付页面，显示支付二维码
3. WHEN 用户完成支付 THEN 系统应更新用户会员状态并发放对应积分
4. WHEN 支付失败 THEN 系统应提示用户重新支付
5. WHEN 用户查看个人中心 THEN 系统应显示当前会员状态、到期时间和剩余积分

**MVP简化说明**:
- 只有2个套餐：免费版（100积分）、基础版（99元/月，1000积分）
- 支付方式：支付宝扫码、微信扫码
- 不需要自动续费
- 不需要发票功能

---

### 需求 3: 选题库

**用户故事**: 作为内容创作者，我想要浏览热门选题，以便快速找到适合我的内容主题。

#### 验收标准

1. WHEN 用户访问选题库页面 THEN 系统应显示30个精选选题列表，包含标题、热度、分类
2. WHEN 用户点击选题 THEN 系统应显示选题详情，包含内容建议、标题方向、爆款案例
3. WHEN 用户点击"立即创作" THEN 系统应跳转到文案生成页面并自动填充选题信息
4. WHEN 用户收藏选题 THEN 系统应将选题添加到收藏列表
5. WHEN 用户选择分类筛选 THEN 系统应只显示该分类下的选题

**MVP简化说明**:
- 选题内容由管理员手动维护（30个精选选题）
- 分类固定为5个：知识教育、职场商业、生活技巧、美食探店、其他
- 不需要搜索功能
- 不需要AI生成选题

---

### 需求 4: AI文案生成

**用户故事**: 作为内容创作者，我想要使用AI生成文案，以便快速获得高质量的短视频文案。

#### 验收标准

1. WHEN 用户输入内容主题 THEN 系统应显示配置选项（平台、风格、时长）
2. WHEN 用户点击"生成文案" THEN 系统应调用大模型API生成3个版本的文案
3. WHEN 文案生成完成 THEN 系统应显示3个文案版本，包含标题、正文、标签
4. WHEN 用户选择一个版本 THEN 系统应允许用户编辑文案内容
5. WHEN 用户保存文案 THEN 系统应将文案保存到草稿箱并扣除5积分

**MVP简化说明**:
- 固定消耗5积分/次
- 平台选项：抖音、小红书、视频号
- 风格选项：知识干货、搞笑幽默、情感共鸣
- 时长选项：30-60秒、1-3分钟
- 使用GPT-4 Turbo API

---

### 需求 5: 数字人视频生成

**用户故事**: 作为内容创作者，我想要生成数字人视频，以便快速制作短视频内容。

#### 验收标准

1. WHEN 用户完成文案后点击"生成视频" THEN 系统应显示数字人形象选择界面
2. WHEN 用户选择数字人形象和配置 THEN 系统应显示预估消耗积分（20积分）
3. WHEN 用户确认生成 THEN 系统应调用数字人API开始生成视频
4. WHEN 视频生成中 THEN 系统应显示进度条和预计剩余时间
5. WHEN 视频生成完成 THEN 系统应显示视频预览和下载按钮

**MVP简化说明**:
- 固定消耗20积分/次
- 提供5个预设数字人形象
- 配音类型：数字人原声、AI男声、AI女声
- 背景类型：纯色、虚化
- 视频比例：9:16竖屏、16:9横屏
- 字幕：默认开启
- 使用HeyGen API

---

### 需求 6: 内容管理

**用户故事**: 作为内容创作者，我想要管理我的创作内容，以便查看和下载已生成的视频。

#### 验收标准

1. WHEN 用户访问"我的内容"页面 THEN 系统应显示所有已创作的内容列表
2. WHEN 用户点击内容 THEN 系统应显示内容详情，包括文案和视频
3. WHEN 用户点击下载 THEN 系统应下载视频文件到本地
4. WHEN 用户删除内容 THEN 系统应从列表中移除该内容
5. WHEN 用户访问草稿箱 THEN 系统应显示所有未完成的内容

**MVP简化说明**:
- 只支持下载，不支持自动发布到平台
- 不支持定时发布
- 不支持批量操作

---

### 需求 7: 个人中心

**用户故事**: 作为内容创作者，我想要查看我的账号信息，以便了解会员状态和积分余额。

#### 验收标准

1. WHEN 用户访问个人中心 THEN 系统应显示个人信息、会员状态、积分余额
2. WHEN 用户点击"编辑资料" THEN 系统应允许修改昵称和头像
3. WHEN 用户查看使用记录 THEN 系统应显示积分消耗历史
4. WHEN 用户查看订单历史 THEN 系统应显示所有支付订单
5. WHEN 用户点击"升级会员" THEN 系统应跳转到套餐页面

**MVP简化说明**:
- 不需要详细的数据统计
- 不需要账号设置（如修改密码、绑定手机等）

---

### 需求 8: 响应式设计

**用户故事**: 作为内容创作者，我想要在不同设备上使用系统，以便随时随地进行创作。

#### 验收标准

1. WHEN 用户在桌面浏览器访问系统 THEN 系统应显示完整的多列布局界面
2. WHEN 用户在移动设备访问系统 THEN 系统应自动调整为单列布局
3. WHEN 用户调整浏览器窗口大小 THEN 系统应动态调整布局
4. WHEN 用户在移动端操作 THEN 系统应优化触摸交互体验
5. WHEN 界面元素重新排列 THEN 系统应保持所有功能的可访问性

**MVP简化说明**:
- 优先保证桌面端体验
- 移动端做基础适配即可

## MVP用户旅程

```
1. 注册/登录
   ↓
2. 浏览选题库，选择一个选题
   ↓
3. 点击"立即创作"，进入文案生成页面
   ↓
4. 配置参数（平台、风格），生成文案（消耗5积分）
   ↓
5. 选择满意的文案版本，可编辑
   ↓
6. 点击"生成视频"，选择数字人形象
   ↓
7. 配置视频参数，生成视频（消耗20积分）
   ↓
8. 预览视频，下载到本地
   ↓
9. 手动发布到抖音/小红书等平台
```

## MVP不包含的功能

以下功能将在V2.0版本中实现：

1. **AI分身功能** - 学习博主风格，爆款洗稿
2. **工具市场** - 文案提取、文案二创、竞品分析、视频拆解
3. **自动发布** - 直接发布到抖音/小红书/视频号
4. **数据分析** - 内容表现统计、趋势分析
5. **账号管理** - 绑定社交媒体账号
6. **团队管理** - MCN机构多账号管理
7. **管理后台** - 工作流管理、用户管理
8. **高级功能** - 定时发布、批量操作、自定义数字人

## 成功标准

- 从选题到生成视频的完整流程 < 5分钟
- 文案生成时间 < 10秒
- 视频生成时间 < 90秒
- 系统响应时间 < 2秒

- 注册用户 > 500 (第1个月)
- 付费用户 > 50 (第1个月)
- 付费转化率 > 10%
- 用户留存率 > 40% (7日)

### 技术指标
- API成功率 > 95%
- 系统可用性 > 99%
- 页面加载时间 < 3秒
